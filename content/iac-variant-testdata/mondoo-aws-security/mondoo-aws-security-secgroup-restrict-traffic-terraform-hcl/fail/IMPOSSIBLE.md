The check only selects security groups that contain an ingress block with `arguments.to_port == -1` (the "all ports" sentinel), then asserts `.none(... cidr_blocks.contains("0.0.0.0/0") ...)`. The terraform-hcl provider cannot represent a numeric `-1` in this position: the HCL literal `to_port = -1` parses to `1.0` (the unary minus is dropped), `0 - 1` splits into a two-element list `[0, 1]`, and `"-1"` yields a string. So `.where(blocks.where(type=="ingress").contains(arguments.to_port == -1))` is always empty, `.none(...)` is vacuously true, and the check always scores 100 for any HCL input. No failing fixture is possible.
