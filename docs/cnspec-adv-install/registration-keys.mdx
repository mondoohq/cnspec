cnspec must be registered with Mondoo Platform in order to download policies and report results. Typically you [register cnspec using a temporary token](registration). However, to ease automation, you can use a client credentials file instead. A common example is when you're integrating cnspec into CI/CD pipelines.

<Callout type="warn">
  The credentials file includes a private key. Keep this credential safe!
</Callout>
1. Log into the [Mondoo Console](https://console.mondoo.com).

2. [Navigate](/start/navigate/) to the [region](/start/organize/regions), [organization](/start/organize/organizations), and [space](/start/organize/spaces) where you want to see the new asset's scan results.

3. In the left navigation, under **Integrations**, select **Add New Integration**.

4. Under **Supply Chain**, select **Generate Long-Lived Credentials**.

5. Select the **GENERATE NEW CREDENTIALS** button.

6. Copy the JSON string and save it in a .json file.
